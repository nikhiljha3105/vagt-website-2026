## 2. Backend API — All Endpoints

### `/api/auth/...` (no auth required)
```
POST /api/auth/employee/register        Guard registration step 1 (name/phone/email)
POST /api/auth/employee/verify-otp      Guard registration step 2 (creates disabled Auth account)
POST /api/auth/employee/resend-otp      Resend registration OTP
POST /api/auth/guard/keycode-login      Keycode → Firebase custom token
POST /api/auth/forgot-password          Initiate password reset OTP
POST /api/auth/reset-password           Verify OTP + set new password
POST /api/auth/resend-reset-otp         Resend reset OTP
```

### `/api/...` (employee role)
```
GET  /api/attendance/today              Today's check-in status

POST /api/employee/incidents            File new incident
```

## 4. Known Bugs & Incomplete Workflows

### Bugs (things that exist but are broken)

| Severity | Where | What |
|----------|-------|------|
| High | `admin-portal.html` | Leave approve/reject writes directly to Firestore — bypasses the API that decrements leave balance. Balance is never updated when admin approves via dashboard. |
| High | `admin-portal.html` | Guard registration approve/reject also bypasses the API — the API does more (enables Auth account, sends reset email, creates employee doc). Direct Firestore write does none of that. |
| Medium | `admin-admins.html` | Password input is `type="text"` — shows plaintext as you type |
| Medium | `employee.js` | `todayStr()` uses UTC not IST — after 18:30 IST (midnight UTC) date rolls over early. Guards on night shift checking in after 18:30 get logged under tomorrow's date. |
| Medium | `auth.js` | SMS not wired — registration OTP is stored in Firestore but never sent. Must read manually for testing. |
| Low | `index.js` | `SETUP_PASSPHRASE` defaults to `null` if env var not set — first-admin setup endpoint silently fails with "wrong passphrase" |
| Low | `activity_log` | No TTL or archival — will grow unbounded |

### Incomplete Workflows

| Workflow | What exists | What's missing |
|----------|-------------|----------------|
| Guard onboarding | Register → OTP → pending doc → admin sees it → approve/reject API | SMS delivery (MSG91), password-reset SMS to guard after approval |
| Daily reports | API endpoint exists, seed data added | No employee-facing page to submit a daily report |
| Payslip PDF download | API endpoint generates PDF | No admin UI to trigger single-slip download; no link from employee payslips list to actually download |
| NFC checkpoint registration | Admin API + `patrol_checkpoints` collection | No admin UI to register a new checkpoint — must call API directly |
| Client invoice PDF | `pdf_url` field exists in schema | PDF generation not implemented, field always null |
| Custom domain | Code ready | `vagtservices.com` not connected to Firebase Hosting |
| Multilingual UI | Architecture noted | Not started |

### Not Built At All
- Guard daily report submission page (`employee-reports.html` or similar)
- Admin NFC checkpoint registration UI (`admin-patrol.html` exists for viewing logs but not registering new checkpoints)
- LoRa mesh patrol system (student team project — spec exists in repo)
- MSG91 SMS integration
